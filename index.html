    National Aerospace Laboratories

    Bangalore, India

    January 2016 - May 2016

    Introduction

    The present day visual inspection of the aircraft is purely skill based, is time consuming, raises safety issues to the inspector, and there is also lack of the record of the inspection. The accuracy can be affected by inspector's boredom level, weather conditions and weariness. Images captured from the aircraft surface and presented to the inspector after enhancing the defects can make the visual inspection easier and accurate which in turn results in saving time, and ensures the safety of aircraft.

    Fringe pattern detection

    Problem Statement

    Development and implementation of algorithms for the detection of surface defect (dents) in aircraft surfaces using image processing. It involves image acquisition, noise removal, background removal, phase wrapping, phase unwrapping and height mapping.

    Objectives

    The project aims to create a semi-automated system to perform dent detection on the surface of aircraft structures. The objectives include identification of dented regions; classification of the dents from surface features such as writings, markings, textures and patterns; and quantification of the detected dent.

    Methodology

    Use of 2D image processing using edge detection is not good enough for the detection of dents as dents deform the surface into the third dimension. The use of edge detection will also detect the surface features like writing, markings on the surface also as defects. This problem can be solved by trying to re-create the height information of the surface. Variety of ways are described in literature to carry out the profile mapping using optical profilometry. Fringe pattern based optical profilometry method is adopted in this study using 3-step Phase Shift which involves following steps:
    1. Digital fringe projection
    2. Image acquisition
    3. Image processing
    4. Fringe analysis
      • Phase detection
      • Phase unwrapping
    5. Height mapping
    Digital Fringe Projection involves projecting a fringe pattern onto the object surface. A sinusoidal fringe pattern is used as it has features that counter defocussing and is most apt to be processed by linear filtering. The setup consists of a projection unit, an acquisition unit and a processing or analysis unit. The images of the reference plane as well as of the object are captured with the camera, perpendicular to the surface of the plane or object, and the projector at an angle in the Image Acquisition step. The captured images contain the projected patterns on the reference and object surface. 3 phase shifted images are acquired per surface.

    Stages of dent detection

    Final result: Dent depth

    The images that are captured include the surface that needs to be tested as well as certain unwanted elements, such as the background, shadows and reflected light. Image Processing involves the pre-processing of the captured images to remove the aforementioned unwanted elements. Background removal is performed to improve computation time and avoid redundant processing. Fringe Analysis is the next operation performed on the pre-processed images. Fringe Analysis consists of 2 steps: (i) phase detection, and (ii) phase unwrapping. In phase detection, the image is analysed to produce a wrapped phase map distribution. An inverse tangent function is used to generate phase map from the intensity values of the captured images. The resultant map contains the localised phase values of the pixels. The recovered/estimated phase is obtained in the interval [- π, + π]. In general, true phase may range over an interval greater than 2π. In such cases, phase values contain artificial discontinuities. The process of determining the unknown integral multiple of 2π to be added at each pixel of the wrapped phase map to make it continuous by removing the artificial 2π discontinuities is referred to as phase unwrapping. Phase unwrapping is an important step with many pre-existing algorithms designed to perform the task. The unwrapped phase map obtained contains the true phase values at each pixel location. The phase values are then converted into depth information using Height Mapping. The dent is detected based on the height map, and then quantified to find the relative dent depth and area of the dented region. Calibration is needed to perform accurate image to real world coordinate transformation.
    Unit testing, Integration testing and system testing were done to see if if all the modules are working correctly, individually and when they are integrated with other modules. Testing will also help remove bugs from the software. After testing all the modules, user interface was developed and the final touches to the software were made.

    Publication

    We also published a paper based on this research:
    Augustin M.J, Archana Ramesh, Yaswi Jalan, Priyank Singh, Nitin Joy, Amit Gupta, Nitesh Gupta, “Adaptive Background Removal for Fringe Profilometry based Surface Inspection of Aircraft Structures”, National Seminar on Aerospace Structures XIX, February 23-25th, 2017, VIT University, Vellore India

    • Usability Testing for the Bergen Swamp Preservation Society website

    RIT, Rochester, NY

    January 2017 - May 2017

    Objective

    The goal was to perform usability tests on the Bergen swamp preservation society website to identify the usability issues that existed in it. The main aim of the study was to make sure that the users could clearly understand and use all the features of the website.

    Purpose

    We were a team of 6 for this project, and we collaborated with the developer of the new Bergen Swamp Preservation Society website to go about with our usability studies. The main goal was to see if the user experience in the website mapped to the users conceptual model on how to use the website. The purpose was to detect all the usability issues related to the website and also get feedback from all the participants on the website and suggestions on how to improve the website.

    Stakeholders

    Teenagers to people aged 60+ were our target audience.

    Process

    The Bergen swamp preservation society website had many functionalities in it like getting directions, calendar, events, photos and so on.
    The first step in our usability testing study was to perform the heuristic evaluations on the website. Initially, all of the members in the group, individually used the different heuristics and all the gestalt principles to detect the usability flaws in the website; and we then merged all of our findings.
    We created flyers to recruit 5 students from Rochester Institute of Technology for the study. All potential participants had to take the screener questions, using which we filtered the participants to select the 5 eligible students who visited websites related to nature and nature reserves. A 20$ Amazon gift card was given as a reward after the study to all the participants. Pilot tests were performed initially with 2 participants, using which test design was made and a test plan was formulated.
    5 tasks were created and all of them were based on problem scenarios. These tasks were shuffled between all participants so that there is no transfer of knowledge between tasks and the tasks were counterbalanced.
    The success criteria for each task was predetermined and decided upon by all the team members in advance. Qualitative analysis was performed using the data collected by asking the participants questions after each task, and Quantitative analysis was performed using the data collected by counting the number of clicks and the time taken for every task.
    SUS scale was used to measure the usability of the system.

    Setup

    The usability tests were performed in the usability lab at B. Thomas Golisano College of Computing and Information Sciences. There were two rooms, a testing room and an observation room. There was a one way mirror between them, where only the observers in the observation room could see the people in the testing room. The morae software (Morae-observer and Morae-observer) was used to record all the observations of the observers. There was a web camera and microphone used to record and listen to the participants. Based on the input, feedback and suggestions made by the participants; a report was written. We presented the whole usability study to the client and submitted the report with the recommendations and usability issues present in the website.

    Results

    Based on the intensive usability studies conducted on the Bergen Swamp Preservation Society website and the results gathered, the following suggestions were made:
    • To replace the property icons with pictures or linking those icons to Maps/Photos.
    • Segregating the photos in the photo gallery as to which properties was featured in the photograph.
    • Making the “Become a Member”, “Visitor Request” more prominent in the main tab.
    • Including the ‘Visitor Policy’ data in the ‘Visitor Request’ form or having the link to ‘Visitor Request’ form in the ‘Visitor Policy’ page.
    • Adding more information about becoming a Member and what each membership type entails.
    • Adding more information about the Swamp itself.
    • Renaming the Favorite Links section as “Resources” or “Links” or moving it higher up on the homepage where it is easily visible.
    • Moving the membership details to the ‘About us’ or to the ‘Contact Us’ section.

    Accoutre

    RIT, Rochester, NY

    January 2017 - May 2017

    Introduction

    Accoutre is a mobile application for dressing, that is aimed at providing suggestions and allowing its users to pick out clothes, either
  • based on the current weather outside, or the weather throughout the day, or/and
  • based on an occasion or an event that they are to attend, such as a party, wedding, interview, sports activity, and so on, scheduled on their calendar or in their emails.
    • All the major designing of the app was done using Adobe Photoshop.

    Stakeholders

    Primary stakeholders of our application would be the end users who own and use a smartphone, and ones who need help in picking out clothes based on the current weather outside, or an event they are to attend.

    Contextual Inquiry

    We conducted contextual inquiries for 12 participants in total. The interviews were conducted via video calls, or in person. We interviewed both male and female participants, between the ages of 20 to 60+ years, and who owned a smartphone. To ensure that the data we collected from our contextual inquiries came from a relatively diverse set of end users, each of our group members interviewed participants who were located at different geographical regions, with varying climates and weather conditions. The interviews also provided us with different scenarios for the type of occasions or events that the participants were most likely to attend or dress for.
    We gathered important information about the factors influencing the daily routine of picking out clothes for our participants. Most users used a weather application on their smartphones to check the weather, and then proceeded with selecting their clothes. While a few users, mostly college students, picked out their clothes randomly. We also came across a user who took suggestions from their friends and family, to select their outfit for a special occasion or event. The interviews also provided us with different scenarios for the type of occasions or events that the participants were most likely to attend or dress for.

    Design Ideation and Brainstorming

    From the data collected through our contextual inquiries, we could identify the primary stakeholders of our application, that is, the end users who own and use a smartphone, and ones who need help in picking out clothes based on the current weather outside, or an event they are to attend. Based on the information gathered from our study models – flow models, sequence models, artifact models, physical models, and cultural models, we consolidated our findings into an affinity diagram, and a consolidated sequence model. In general, our key observations from ‘walking the wall’, with respect to the primary workflows for users to pick out their clothes were as follows:
  • Users check the current weather outside or the weather throughout the day, using a reliable weather app, before picking out clothes for that day.
  • They select clothes based on an occasion or an event, such as a party, wedding, interview, sports activity, and so on.
  • They pick out clothes randomly.
  • They use applications such as Instagram or Pinterest to follow fashion trends and be aware.

    • Lo-Fi prototype

    Specific Tasks and App Features

    To narrow down on the design ideas for our application from the users' perspective, we identified the following tasks and features as they relate to the goals:
  • Sync with the calendar application.
  • Access to user’s current location.
  • Access and connect to the weather application.
  • Sort clothes based on categories, for example – type, fabric or season.
  • Option to feed-in clothes present in user’s current wardrobe under each category.
  • Run algorithms to choose and suggest clothes based on weather, events, or fashion trends.
  • Option to choose and mix-and-match clothes manually, in case the algorithm gets it wrong.
  • Suggestion to shop for clothes online, based on current requirement and unavailability in user’s closet.
  • Suggestion of suitable online retailers, and direction to their official websites.
    • Based on the information gathered, we have generated the various study models, and have consolidated our findings into an affinity diagram, and a consolidated sequence model. We were successful in gathering sufficient information to help us understand users’ patterns of picking out clothes, which would help us tremendously in the next phase of our project.

    Hi-Fi prototype

    Future Improvements

    For future work, the aspects of relevant tasks and activities that could be improved are as follows:
  • The application would help users to conjure a virtual image of how they would look in an outfit.
  • It would allow users to connect with their friends/family, share images of outfits with them, and take suggestions.
  • It would notify users about online offers or sales by their favorite retailers, or based on their current wardrobe requirement.
  • It would allow users to save and maintain their online shopping history.

    • Problem Scenario

    Each morning, Jenny wakes up and checks the calendar application and e-mail notifications on her phone. She then takes a shower, and randomly picks out clothes to wear to work. Since her office is 10 minutes away from her home, she usually walks to work. One day, on her way to work, it started to pour suddenly. Because Jenny was not prepared for the rain, and did not have a raincoat or an umbrella with her, her clothes and bag got completely drenched.

    Design Scenario

    Ever since Jenny started using the Accoutre app, she wakes up each morning and checks the app, along with other notifications on her phone. She then takes a shower, and wears the clothes suggested by the Accoutre app, based on the current weather outside, and/or an occasion or event she must attend, like a boarding meeting or a party. The app helps her pick out the appropriate outfit each time.

    Process

Next, personas were created for each profile based on the data acquired from the contextual inquiries and field observations.
    The final step consisted of the creation of Problem and Design Storyboards. The design Storyboard/ Scenario would help us determine how the design improvement we have suggested would affect the problem scenario. Both text and graphical representation of storyboards were created.

    Story boarding (Problem)

    Story boarding (Design Solution)

    Results

    1. To add way points.

    There were also some differences in the way people belonging to older age groups used Google maps as compared to the way the younger generation used Google maps. The younger generation of people were acquianted with Google maps and all its features fairly easily compared to the older generation.




    Heuristic Evaluation of Virginia's EZ pass
    system's official website

    RIT, Rochester, NY

    February 2017 - May 2017

    Introduction

    The Aim of the project was to perform heuristic evaluation of the Virginia’s EZ Pass website.

    Process

    The initial step was to perform heuristic evaluation on the website, and all the members of the team performed heuristic evaluation individually to identify the usability and accessibility issues in the website. We then discussed our findings and compiled them into a document which in total had 27 usability and accessibility issues found in the website.

    Results

    After evaluating the Virginia E-ZPass website, we concluded that it had various usability and accessibility issues that need to be fixed. Although some effort has been made to make this website accessible, including the use of 'alt' tags, a 'skip to main content' link and closed captioning for videos; there is more work that needs to be done to make this site fully accessible, especially to those using screen readers. This website provides a valuable service to the drivers and passengers in Virginia, and should be improved to make E-ZPass tasks easier to accomplish for all users.

    Coaster Crunch

    RIT, Rochester, NY

    September 2017 - December 2017

    Introduction

    The most common problem that big amusement parks face today is the huge crowd which lead to long wait times for the rides in the park. The wait times can be so long that people end up getting frustrated and they wish to never return to such amusement parks. Coaster Crunch app solves this problem by introducing a digital fast pass system as well as other additional features.

    Problem

    We all want to spend our leisure time with family and friends especially when we're loaded with work. A theme park is often one of the most favourite places for people to relax, have fun on the rides and forget their troubles for a brief period of time. But isn’t it exhausting to wait in a seemingly never ending queue to get on every single ride? During peak seasons there are queues which could go on up to an hour or even more for every ride. This could be a frustrating problem to a lot of people who have spent a lot of money on the tickets and don’t get to go on all or most of the rides in the park. The whole experience of waiting in a queue for an hour or more for a ride that lasts a few minutes can be stressful, exhausting and demoralizing at times.

    Solution

    A solution for this would be to introduce flash passes or in other words to introduce points for every ride. For instance, let’s say we have one point per ride, and the people have to scan they phone with the barcode for every ride. After two rides, the person can get a fastpass for the next ride, that he has queued up, for which there would be there would be a separate queue where only the people who have accumulated the points and who have that ride queued up for themselves will be able to enter. In this way people would be able to complete most of the rides in an amusement park. The concept of a flash pass has already been initiated by Disney for their amusement parks, however an application does not exist which makes it very exhausting for the people to walk up to each ride, get a pass for that ride and come back at the time that is mentioned on the pass. However, our objective is to digitize the entire concept so we can develop an app. The application will keep track of all the points accumulated by the park visitors which can be used at any time during the year.     Voice assistants are widely used these days, and has made some of our tasks very easy by providing us the ability to interact with devices over voice.     Results

    The qualitative analysis states that the study cannot definitively confirm whether Voice Assistants improve Rochester Institute of Technology student’s academic productivity. Due to the technological limitations of mobile artificial intelligence, as well as the limited available functionality for academic applications, Voice Assistants cannot yet eclipse the traditional modes that students use to stay productive at school. Although the initial impetus for this research bared insubstantial results, this work has provided a general view of the factors that affect the usage of voice assistants with regards to academic productivity.
    In the quantitative study which sought to see impact of Voice Assistants on the academic productivity of international students, research was going to be done to see whether Voice Assistants limit the everyday production of non-Native English speaking students at Rochester Institute of Technology. Our research concluded that the areas which ultimately impact the productivity of international students at Rochester Institute of Technology is the language they choose to communicate with their Voice Assistants, and not their international status. However, we have found that the majority of Voice Assistant users view accents and general language recognition as the biggest issues of Voice Assistant, regardless of their international status and preferred Voice Assistant language. Although we expected that different selections of preferred languages would lead to different user experiences, there are no substantial associations between language choices and dissatisfied response times and accuracy, specifically amongst international students at Rochester Institute of Technology.

    VR App Usability Study

    RIT, Rochester, NY

    July 2018 - December 2018

    Introduction

    The capstone project focused on evaluating a virtual reality application designed for the purpose of watching movies together virtually, by performing a comparative usability study with the video call / screen share approach using TeamViewer software. A usability study was performed for each of the applications, and its results were analyzed to determine the overall advantages, drawbacks, and required feature improvements.

    Method

    Test plans and tasks were created and formulated. The tasks were divided based on the two approaches being used in the study: TeamViewer tasks & Virtual reality tasks
    Counter-balancing of tasks was performed in order to mitigate the issue of transfer of learning. Once tasks were formulated, a pilot test was conducted to validate the tasks and based on the results of the pilot tests, the tasks were fine tuned.
    26 participants, aged between 15 to 80 years, were recruited for the usability studies being performed. These participants remained the same for both TeamViewer and virtual reality study (within-subjects study).
    The entire study lasted for around 30 - 45 minutes each, after which the raw data collected from the interviews was tabulated in an excel worksheet.
    The environment consisted of two rooms, such that voices from one room couldn’t be heard in the other room. Each room had one laptop, one android phone, and 1 VR headset. Each laptop had TeamViewer installed in it, and each android phone had Firefox browser installed in it. A link to a 3 minutes long video clip on YouTube was bookmarked on the browser of each laptop for the participants to watch on TeamViewer. Another 3 minutes video clip was preset in the virtual reality application. The link to the web based virtual reality application was bookmarked in the Firefox browser on the android phone. The android phone was placed in the VR headset as it served as the headset’s screen.

    Findings

    To analyze this purely qualitative data, the responses for both sets of tasks were coded in order to extract common patterns from the raw data. Inductive coding of the raw data was performed by assigning codewords to each of the user input noted for each of the question for each participant. A set of codewords and their counts were maintained, and new code words were added to the set based on the responses discovered during the analysis. Each of the user inputs was assigned code words and two other excel worksheets were maintained to keep track of the code words and the count of those code words for each type of task (TeamViewer and virtual reality application). To avoid multiple counts of a code word for the same participant, a master list of code words for each type of task was maintained per participant, which contained only unique code words and the counts of the code words were incremented using this list. This ensured that if a participant had the code COMMUNICATION_GOOD assigned to more than one response, only total of one occurrence of that code was counted for that participant.

    Click to download the Capstone Research Paper

    Conclusion

    The study resulted in a list of findings stating the positives and negatives for each approach for watching movies together online via two video applications, TeamViewer and Virtual Reality. Overall, the participants had a much better experience with TeamViewer as compared to their experience with the virtual reality application. Even virtual presence of the other participant was felt more using TeamViewer than the virtual reality application. Prior studies found that the use of virtual reality for watching movies was not a good choice because the social presence was not felt as the users had to make an effort to communicate in order to keep the contact during the movie. This study found that the major reasons why virtual presence was not felt using the virtual reality application were the poor or no communication channel, lot of lag in the application making it difficult to focus and causing dizziness / headache, difficulty pausing / playing the movie, square shaped avatar instead of a human-like avatar, discomfort caused by VR headsets, etc. If these issues are fixed, then the virtual reality application seems to have a great potential and would be a great way to watch movies online together. 19 The results were more or less balanced and there was no clear winner, with TeamViewer getting 54% of the total votes. The positive aspects of TeamViewer were that the virtual presence was felt by many participants, since they could communicate with the other participant using audio and webcam video, it was easy to stop/play the movie, however the experience was not immersive and webcam screen was very small. Though virtual reality application received a lot of feedback containing complaints regarding several existing issues like audio, and video lag and the dizziness caused by the lag, communication channel not being established, hard to focus on the pause/play button, cube-shaped avatars making participants feel no virtual presence, and other avatars blocking the view of the movie screen. On the other hand, participants really liked the immersive experience of the virtual reality application which made them feel like watching the movie in an actual theatre and also it was easy for them to move around in the virtual environment using the viewpoints. Participants enjoyed exploring the virtual reality application and the idea of watching a movie together online using the virtual reality application was welcomed with great enthusiasm by most of the participants. The participants who chose TeamViewer as a preference, did so because of the existing issues in the virtual reality application; and they expressed the desire to revisit an improved version of the application in the future. Participants expressed that the virtual reality application had a great potential to be the best method of watching movies virtually, and with improvements it would surely be preferred more than the TeamViewer method.
